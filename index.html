Smarter Regressions

Whilst keeping overnight regressions to serve as an overarching safety net, we've paved the way for smart regressions with the addition of our soon to be released, smart tags feature.

Mobirise Website Builder

You're in control

By adding smart tags you will be able to run UI tests in ways that were previosuly not possible, giving you complete control over execution and allowing you to target the changes that matter most.

Add tags

Add tags to UI tests

Add as many tags as you want to give you granular control over how you would want to run your UI tests. No limits to the amount of tags you can add.

Search by tags

Search for tagged UI tests

ALM are working on giving you the option to search for UI tests by a collection of tags. Again, no limits on how many tags you can search for.

Run by tags

Run UI tests grouped by tags

Once the UI tests matching your tags are returned, simply run them and wait for the results. You'll also be able to manage your tags via ALM.

Step 1

Chromatid will run a script to add the default 'Regression' tag to all UI tests currently in the smoke suites. This will be used for overnight executions.

Step 2

Chromatid will update the Jenkins configuration so that it will no longer require smoke suite ID's and will instead look for the Regression tag.

Step 3 

We'll switch the nightly executions to run based on tags instead of smoke suites. No more having to give us suite ID's.

Phase 2

ALM will soon provide you with the ability to search for and run your UI tests by all other tags you chose to add to your UI tests.
